# v0.3 Release Notes

Harper cluster running v4.7.x (tested on demo.openclaw.harperfabric.com).

## What shipped in v0.3

1. **Incremental cursor persistence for date-granular sources** — `Pipeline` schema now has `cursorTimestamp: String` field to track the last-seen record timestamp. For sources with date-only filtering (USAspending, etc), pipelines now use the persisted cursor as the lower bound instead of fetching the whole day repeatedly.

2. **Troubleshooting entry for stale boot file** — added docs/troubleshooting.md entry for the `HarperDB config file validation error: Specified path database does not exist` symptom and its fix.

3. **First-class `.agents/prompts/` directory** — release-cut playbooks and orchestration prompts now documented in README.md as the canonical place to store per-release instructions.

## What was **not** shipped

- `@createdTime` / `@updatedTime` schema directives — these are Harper v5 features. The cluster runs Harper v4.7.x and does not recognize v5 directives. This is a v0.4+ feature requiring cluster upgrade.

## Verification

- bootstrap.sh passes all 8 steps against live cluster
- USGS example deploys and runs end-to-end
- Cursor persistence confirmed: POST /UsgsEarthquakesRun records 5 earthquakes, GET /Pipeline/usgs-earthquakes shows cursorTimestamp populated
- All JS files validate with `node --check`

## Next steps (v0.4 planning)

- Upgrade demo cluster to Harper v5 Lincoln
- Re-add @createdTime / @updatedTime directives to harper-base and template schemas
- Remove manual timestamp writes in PipelineRegister.post (directives handle them)
